Quick Start

Let's get your start with assay.it.
These few simple steps explain how to run a first quality check.

Sign Up with GitHub

Please use your GitHub account to sign in with assay.it. It only requires access to your public profile, public repositories and access to commit status on connected repositories. Later, you can upgrade the account with access to private repositories and organizations.

Fork Blueprint

Use blueprint application assay-it/blueprint-suite. Fork it to your own GitHub account and then add to the service workspace.

Fork Blueprint

Confirm Quality

Launch the quality assessment job for this example. It implements a minimal quality assessment using Behavior as a Code.

View Suite

Confirm Quality, Eliminate Risks in your Serverless applications

Quality assurance of serverless applications is more complex than doing it for other runtimes. Engineering teams spend twice as much time for maintaining testing environments and mocking cloud dependencies instead of building a loyal relationship with their customers, assay.it has you covered.
